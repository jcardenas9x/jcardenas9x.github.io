I had immediately decided to springboard off my current experience working with the amazing [GatsbyJS](https://gatsby.org) and use that for the project I plan to build as the capstone for this #100DaysOfCode challenge. As mentioned in my previous post, a big goal I want to achieve is to try and be better about front-end UI and UX design. However, due to the nature of this project, I still have to devise a way to automate data consumption, so specific parts of what I can work on for the mean time is somewhat limited.

For now, I have decided to try and work on establishing some basic styling and simple look-and-feel components. This is also my first time working with [CSS-in-JS](https://cssinjs.org/) as I previously have either used Sass, `styled-components` or a combination of both. For the project I have decided to use [Glamor](https://github.com/threepointone/glamor) to accomplish this, though for now I have mostly just used inline CSS. 

CSS is something I have always struggled with in the past, because I always had trouble visualizing how combinations of properties would appear on the screen. In working on this project I do hope this is something I can get better with, especially due to the increase in tools and utilities in making CSS less of a giant headache to work with.

My major upcoming challenges through the following week:

* Attempting proper APK mining for data consumption - **Critical**
* More elegant CSS styling/design
* Organization of data for consumption
* Data representation
* Navigation
